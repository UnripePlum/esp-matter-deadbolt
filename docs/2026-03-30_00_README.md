## 2. GD-3000 동작 특성 (하드웨어 테스트 확인됨)

### 2-1. 잠금/해제 메커니즘

| 동작 | 방법 | 조건 |
|------|------|------|
| **잠금** | 자동 (GD-3000 내부) | 12V 인가 + 자석 감지(문 닫힘) + 잠금 시간(3초) 경과 |
| **해제 방법 1** | 12V 차단 (릴레이 #1) | 즉시 볼트 후퇴 |
| **해제 방법 2** | EXIT 단락 (릴레이 #2) | 즉시 볼트 후퇴 |

### 2-2. 센서 특성

GD-3000의 NO/COM 센서는 **볼트 위치가 아니라 자석(문 열림/닫힘)**을 감지합니다.

| 센서 상태 | GPIO 5 | 의미 |
|-----------|--------|------|
| NO-COM 단락 | LOW | 문 닫힘 (자석 붙음) |
| NO-COM 개방 | HIGH | 문 열림 (자석 떨어짐) |

### 2-3. 전원 특성

| 항목 | 값 |
|------|-----|
| 대기 전류 | ~300mA @ 12V |
| 동작 전류 | ~800mA @ 12V |
| 정전 시 | 볼트 후퇴 → 해제 (Fail-Safe) |
| 5V만 차단 시 | 릴레이 OFF → NC 단락 → 12V 유지 → 잠금 (Fail-Secure) |

## 3. 모듈별 상세

### 3-1. hal_gpio (하드웨어 추상화)

| GPIO | 기능 | 방향 | 설명 |
|------|------|------|------|
| 4 | 릴레이 #1 (전원 NC) | 출력 | HIGH=12V 차단(해제), LOW=12V 인가(잠금 유지) |
| 5 | 문 닫힘 센서 (자석) | 입력 (풀업) | LOW=문 닫힘, HIGH=문 열림 |
| 6 | 릴레이 #2 (EXIT NO) | 출력 | HIGH=EXIT 단락(해제), LOW=EXIT 개방 |
| 48 | 상태 LED (WS2812) | 출력 | led_driver 사용 |
| ADC CH0 | 12V 전원 모니터링 | 입력 | 분압 R1=100k/R2=10k (미연결) |

**주요 함수:**

| 함수 | 설명 |
|------|------|
| `gpio_init()` | 모든 GPIO + ADC + 안전 타이머 초기화 |
| `deadbolt_set(bool unlock)` | NC 릴레이: unlock=GPIO HIGH(12V 차단), lock=GPIO LOW(12V 인가) |
| `deadbolt_exit(bool activate)` | EXIT 릴레이: activate=GPIO HIGH(EXIT 단락) |
| `is_door_closed()` | 자석 센서: true=문 닫힘, false=문 열림 |
| `read_power_voltage_mv()` | ADC → 분압비 역산 → 전압(mV) |

### 3-2. door_controller (도어 제어 엔진)

**상태 관리 모델:**

```
┌─────────────────────────────────────────────────┐
│  LockState (소프트웨어 추적, g_lock_state)         │
│    UNLOCKED ← unlock 명령 시 (즉시)               │
│    LOCKED   ← lock 명령 + 문 닫힘 + 3초 타이머     │
│                                                    │
│  DoorState (자석 센서, is_door_closed())            │
│    CLOSED ← 자석 붙음 (GPIO 5 LOW)                │
│    OPEN   ← 자석 떨어짐 (GPIO 5 HIGH)             │
│                                                    │
│  자동 잠금: 문 열림→닫힘 전환 + 3초 후              │
│            → g_lock_state = true                   │
│                                                    │
│  비정상 개방: g_lock_state=true + 문 열림 감지      │
│            → DOOR_ERR_FORCED 에러 보고              │
└─────────────────────────────────────────────────┘
```

**`door_execute_with_retry(bool target_unlock)`**
- 해제: 12V 차단 (릴레이 ON) → 즉시 g_lock_state = false → 성공
- 잠금: 12V 복원 (릴레이 OFF) → 문 닫힘이면 자동 잠금 타이머 시작 → 성공
- 센서 폴링 검증 없음 (센서는 볼트가 아닌 문 열림/닫힘만 감지)

**`door_queue_command(bool target_unlock)`**
- 명령 큐 (크기 1, 최신 명령 덮어쓰기)
- 워커 태스크가 큐에서 순차 실행
- 실행 중 새 명령 → 큐에 저장 → 현재 완료 후 자동 실행

**`door_exit_open(uint8_t duration_sec)`**
- EXIT 릴레이 펄스 → 즉시 성공
- 실패 시 전원 제어(릴레이 #1) 폴백
- 자동 잠금 타이머 시작 (3~30초)

**주기적 타이머:**

| 타이머 | 주기 | 기능 |
|--------|------|------|
| door_poll | 200ms | 문 열림/닫힘 폴링 + 디바운스 (3회) + 비정상 개방 감지 |
| autolock | 1회성 | 문 닫힘 감지 후 3초 → g_lock_state = true |
| exit_lock | 1회성 | 퇴실 열림 후 자동 잠금 |

**에러 감지:**

| 에러 | 코드 | 조건 |
|------|------|------|
| DOOR_ERR_FORCED (0x05) | 비정상 개방 | 잠금 상태에서 문 열림 감지 |
| DOOR_ERR_POWER (0x03) | 전원 이상 | 전압 <10V 또는 >13V 연속 3회 |

### 3-3. status_led (WS2812 상태 표시)

| 상태 | 색상 | 패턴 |
|------|------|------|
| 부팅 중 | 파랑 | 느린 깜빡임 (500ms) |
| 커미셔닝 대기 | 파랑 | 빠른 깜빡임 (200ms) |
| 잠금 진행 중 | 주황 | 깜빡임 (2초) |
| 잠금 완료 | 주황 | 고정 |
| 해제 진행 중 | 초록 | 깜빡임 (2초) |
| 해제 완료 | 초록 | 고정 |
| 에러 (비정상 개방) | 빨강 | 빠른 깜빡임 (3초) |
| 에러 (센서) | 노랑 | 깜빡임 (3초) |
| 동작 실패 | 빨강 | 깜빡임 (2초) |

우선순위: 에러 피드백 > 시스템 상태 (부팅/커미셔닝) > 잠금 상태

### 3-4. comm_layer (통신 폴백)

3단계 결과 전달: **Matter → BLE → NVS 저장**

```
report_result(result, attempts)
  ├─ Matter 연결됨? → attribute::update(LockState)
  │   └─ 실패? → BLE 폴백
  ├─ BLE 연결됨? → ble_notify_status_encrypted([status, attempts])
  └─ 둘 다 불가? → save_pending_result_to_nvs() (최대 8건, 재연결 시 자동 전송)
```

### 3-5. ble_server (BLE GATT 서버)

**서비스 UUID:** `0000FF01-0000-1000-8000-00805F9B34FB`

| Characteristic | UUID | 속성 | 용도 |
|---------------|------|------|------|
| Command | FF02 | Write (암호화) | 명령 수신 (HMAC 인증) |
| Status | FF03 | Read, Notify (암호화) | 상태/결과 전송 (AES-GCM) |
| Config | FF04 | Read, Write (암호화) | 퇴실 기본시간 설정 |
| Challenge | FF05 | Read (암호화) | 일회용 nonce 발급 |

**보안:**
- LE Secure Connections + MITM + Bonding (최대 5대)
- AES-128-GCM 암호화 (IV: counter 4B + random 8B = 12B)
- Challenge-Response (HMAC-SHA256, UNLOCK/EXIT_OPEN 인증 필수)
- Constant-time HMAC 비교 (타이밍 공격 방지)

### 3-6. app_main (엔트리 포인트)

**초기화 순서:**
1. GPIO 4 즉시 LOW 고정 (BC547 채터링 방지, 잠금 상태)
2. GPIO 전체 초기화 (릴레이 #1/#2, 센서, LED, ADC)
3. NVS Flash 초기화
4. BLE 암호화 키 로드 (NVS, NimBLE 불필요)
5. Matter 노드 + Door Lock 엔드포인트 생성
6. 도어 컨트롤러 초기화 (명령 큐, 워커 태스크, 폴링 타이머, 자동 잠금 타이머)
7. Matter 시작 (NimBLE 초기화 포함)
8. BLE GATT 서버 등록 (Matter NimBLE 위에)
9. Status LED 초기화 + 시스템 준비 완료

**Matter 명령 처리:**
- `emberAfPluginDoorLockOnDoorLockCommand` → `door_queue_command(false)`
- `emberAfPluginDoorLockOnDoorUnlockCommand` → `door_queue_command(true)`
- attribute callback은 LockState 로그만 출력 (릴레이 조작 없음, 피드백 루프 방지)

**커미셔닝 LED:**
- fabric 유무 + 커미셔닝 창 상태 → `refresh_commissioning_led()`
- `kCommissioningComplete`, `kServerReady`, `kFabricCommitted` 이벤트에서 갱신

## 8. 알려진 제한사항 및 향후 작업

- **AES-128-GCM 키:** NVS 자동생성 방식. 양산 시 기기별 사전 프로비저닝 필요.
- **GD-3000 센서:** 자석(문 열림/닫힘)만 감지, 볼트 위치 감지 불가. 잠금 상태는 소프트웨어 추적.
- **NC 배선:** 릴레이 OFF=12V 인가(잠금), 릴레이 ON=12V 차단(해제). 정전=해제(Fail-Safe), 5V만 차단=잠금(Fail-Secure).
- **자동 잠금:** GD-3000 내부 자석 감지 + 잠금 시간(3초) 설정. ESP32 자동 잠금 타이머와 동기화.
- **전원 모니터링:** 분압 회로 미연결, 비활성 상태.
- **OTA:** 파티션 레이아웃 준비, End-to-End 미테스트.
- **BLE + Matter 커미셔닝:** 코드 완료, 하드웨어 검증 완료.
